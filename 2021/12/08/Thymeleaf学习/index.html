Thymeleaf学习

发表于 2021-12-08

Thymeleaf是一个Java XML/XHTML/HTML5开源的模板引擎，可以在Web(基于servlet)和非Web环境中工作，适合在基于MVC的Web应用程序的视图层提供XHTML/HTML5，也可以在脱机环境中处理任何XML文件，提供完整的Spring Framework(Spring/SpringBoot推荐使用thymeleaf)。在Web应用程序中，Thymeleaf旨在成为JavaServer Pages（JSP）的完全替代品，并实现自然模板的概念：模板文件可以直接在浏览器中打开，并且仍然可以正确显示为网页。

Thymeleaf

Thymeleaf

Docs

1.Thymeleaf 是一个模板引擎,可用于生成用户界面或 View 视图层,以 XHTML 或 HTML 形式( Web 框架: Spring MVC、Struts或Apache Wicket，模板引擎:JSP、Velocity或 FreeMarker)

2.Thymeleaf 处理模板类型

  • HTML (HTML5, XHTML 1.0/1.1, HTML 4)
  • XML
  • Text(纯文本)
  • JavaScript(.js)
  • CSS(.css)

3.使用Thymeleaf可以完全替代 JSP 和 JSTL。

Thymeleaf+Spring

  • Make the mapped methods in your Spring MVC @Controller objects forward to templates managed by Thymeleaf, exactly like you do with JSPs.
  • Use Spring Expression Language (Spring EL) instead of OGNL in your templates.
  • Create forms in your templates that are completely integrated with your form-backing beans and result bindings, including the use of property editors, conversion services and validation error handling.
  • Display internationalization messages from message files managed by Spring (through the usual MessageSource objects).
  • Resolve your templates using Spring’s own resource resolution mechanisms.

SpringStandard方言

该方言基于 Thymeleaf 标准方言，即org.thymeleaf.spring5.dialect.SpringStandardDialect类继承自org.thymeleaf.standard.StandardDialect类.

除了继承标准方言中已经存在的所有特性外，SpringStandard方言还引入了以下特定特性：

1.使用 Spring Expression Language（Spring EL 或 SpEL）作为变量表达式语言，而非 OGNL。因此，所有${...}和*{...}表达式将用 Spring Expression Language engine处理。

2.使用 SpringEL 的语法可访问应用context上下文中的 bean(${@myBean.doSomething()})。

3.用于表单处理的新属性:th:field、th:errors和th:errorclass。

4.新的表达式对象和方法:#themes.code(...)相当于 JSP 自定义标记中的spring:theme,#mvc.uri(...)相当于 JSP 自定义函数的spring:mvcUrl(...)。

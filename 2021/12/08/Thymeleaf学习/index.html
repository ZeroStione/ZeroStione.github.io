Thymeleaf学习

发表于 2021-12-08 更新于 2022-03-19

Thymeleaf是一个Java XML/XHTML/HTML5开源的模板引擎，可以在Web(基于servlet)和非Web环境中工作，适合在基于MVC的Web应用程序的视图层提供XHTML/HTML5，也可以在脱机环境中处理任何XML文件，提供完整的Spring Framework。

Thymeleaf

Thymeleaf

Docs

1.Thymeleaf 是一个模板引擎,可用于生成用户界面或 View 视图层,以 XHTML 或 HTML 形式( Web 框架: Spring MVC、Struts或Apache Wicket，模板引擎:JSP、Velocity或 FreeMarker)。在Web应用程序中，Thymeleaf旨在成为JavaServer Pages（JSP）的完全替代品，并实现自然模板的概念：模板文件可以直接在浏览器中打开，并且仍然可以正确显示为网页，Spring/SpringBoot推荐使用thymeleaf。

2.Thymeleaf 处理模板类型

  • HTML (HTML5, XHTML 1.0/1.1, HTML 4)
  • XML
  • Text(纯文本)
  • JavaScript(.js)
  • CSS(.css)

3.使用Thymeleaf可以完全替代 JSP 和 JSTL。

模板引擎

模板引擎是为了使用户界面与业务数据（内容）分离而产生的，它可以生成特定格式的文档，用于网站的模板引擎就会生成一个标准的HTML文档。

  • 置换型模板引擎(将指定模板内容（字符串）中的特定标记（子字符串）替换,实现简单，但其效率低下，无法满足高负载的应用需求)

  • 解释型模板引擎

  • 编译型模板引擎

初识Thymeleaf

1.Standard Expression Syntax

  • Simple expressions:
    • Variable Expressions: ${...}
    • Selection Variable Expressions: *{...}
    • Message Expressions: #{...}
    • Link URL Expressions: @{...}
    • Fragment Expressions: ~{...}
  • Literals
    • Text literals: 'one text', 'Another one!',…
    • Number literals: 0, 34, 3.0, 12.3,…
    • Boolean literals: true, false
    • Null literal: null
    • Literal tokens: one, sometext, main,…
  • Text operations:
    • String concatenation: +
    • Literal substitutions: |The name is ${name}|
  • Arithmetic operations:
    • Binary operators: +, -, *, /, %
    • Minus sign (unary operator): -
  • Boolean operations:
    • Binary operators: and, or
    • Boolean negation (unary operator): !, not
  • Comparisons and equality:
    • Comparators: >, <, >=, <= (gt, lt, ge, le)
    • Equality operators: ==, != (eq, ne)
  • Conditional operators:
    • If-then: (if) ? (then)
    • If-then-else: (if) ? (then) : (else)
    • Default: (value) ?: (defaultvalue)
  • Special tokens:
    • No-Operation: _

2.Attribute Precedence

Order Feature Attributes
1 Fragment inclusion th:insert th:replace
2 Fragment iteration th:each
3 Conditional evaluation th:if th:unless th:switch th:case
4 Local variable definition th:object th:with
5 General attribute modification th:attr th:attrprepend th:attrappend
6 Specific attribute modification th:value th:href th:src ...
7 Text (tag body modification) th:text th:utext
8 Fragment specification th:fragment
9 Fragment removal th:remove

Spring

  • Make the mapped methods in your Spring MVC @Controller objects forward to templates managed by Thymeleaf, exactly like you do with JSPs.
  • Use Spring Expression Language (Spring EL) instead of OGNL in your templates.
  • Create forms in your templates that are completely integrated with your form-backing beans and result bindings, including the use of property editors, conversion services and validation error handling.
  • Display internationalization messages from message files managed by Spring (through the usual MessageSource objects).
  • Resolve your templates using Spring’s own resource resolution mechanisms.

View

1.SpringMVC模板系统有两个核心接口：

  • org.springframework.web.servlet.View
  • org.springframework.web.servlet.ViewResolver

2.View(视图)负责渲染实际的HTML界面，通常通过由模板引擎处理，如Thymeleaf或FreeMarker。

3.ViewResolvers(视图解析器)负责获取特定操作和语言的View对象的对象。通常，控制器要求 ViewResolvers转发到特定的视图（视图名为控制器方法返回的字符串 如下例的hello），然后顺序执行应用中所有的视图解析器，直到有一个能够解析这个视图（匹配到该视图名对应的实际地址），最终返回一个真正的View对象。
